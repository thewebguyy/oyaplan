# Archive

Documents in this folder are kept for historical reference only. **None of them are authoritative.** If anything here conflicts with `CLAUDE.md`, `docs/product/`, or `docs/decisions/`, those win.

## `ai-roleplay-framework/`

A 13-file "multi-agent AI operating system" narrative (Founder / GPT-as-product-strategist / Claude-as-CTO / "Antigravity IDE"-as-executor) generated by a prior agent session, moved here wholesale rather than deleted.

Reasons it was archived instead of kept live as project documentation:

- **Internal duplication.** `claude_context.md` and `claude_context_package.md` are near-identical; so are `gpt_context.md` and `gpt_context_package.md`.
- **Claims contradicted by the actual codebase.** For example, `engineering_handoff.md` asserts "no strict unit test suite... is currently mandated," while `docs/decisions/0002-vitest-as-test-runner.md` (and the real `.github/workflows/ci.yml`) show a working Vitest suite gated in CI. `executive_baseline.md` describes the AI orchestrator as a completed platform pillar; the actual `lib/services/ai/` module is a fully-mocked stub (`MockProvider`, no LLM SDK dependency, zero real importers anywhere in the app).
- **Claims contradicted by the approved product strategy.** `gpt_context_package.md` states "Rejected Idea: Forcing login to view plans" as settled product direction — the opposite of `docs/product/mvp-scope.md` §2.1/§2.6, which requires mandatory phone/OTP signup.
- **A self-issued, inaccurate "all clear."** `ai_synchronization_report.md` declares "100% Reconciled... Zero unresolved contradictions remain." That was not true when written and is not true now — see `docs/product/README.md` for the conflicts that are actually open.
- **A governance layer with no current operative counterpart.** The "Founder / GPT / Claude / Antigravity IDE" role hierarchy in `ai_communication_protocol.md` and the two `prompts/*_operating_prompt.md` files does not correspond to how this repository is actually operated (a single engineering charter, `CLAUDE.md`, governs AI-assisted work here — there is no separate "GPT" product-strategy actor with veto power in practice).

Keep this folder if you want the history of how the documentation drifted; do not treat any claim in it as current fact without independently checking the codebase or `docs/product/`.
